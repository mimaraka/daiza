# Daiza

เว็บแอปที่วิเคราะห์จุดศูนย์ถ่วงของฟิกเกอร์อะคริลิคจากภาพ PNG และคำนวณ **ตำแหน่งร่องเสียบ** และ **ขนาดฐาน** โดยอัตโนมัติ ส่งออกเส้นตัด (เส้นรอบรูปอะคริลิค) และแบบฐานเป็น SVG ในระบบพิกัดขนาดจริง (มม.)

**URL สาธารณะ: https://mimaraka.github.io/daiza/**

การวิเคราะห์ภาพ การคำนวณเรขาคณิต และการสร้าง SVG ทำงานทั้งหมดภายในเบราว์เซอร์ **ภาพที่โหลดและข้อมูลการวิเคราะห์จะไม่ถูกส่งออกไปภายนอก** (เป็นเว็บไซต์สแตติกที่ไม่มีเซิร์ฟเวอร์)

## คุณสมบัติ

- **การวิเคราะห์จุดศูนย์ถ่วง** — ถือบริเวณที่ไม่โปร่งใส (ค่าเริ่มต้น: α>0; เกณฑ์อัลฟาจะเปลี่ยนขอบเขตการตัดสินใจ) เป็นพื้นที่ความหนาแน่นสม่ำเสมอ และคำนวณจุดศูนย์ถ่วงของพื้นที่ที่ล้อมรอบด้วยเส้นตัด
- **การจัดวางร่องเสียบอัตโนมัติ** — วางโครงสร้าง 2 ชั้น "คอ + ลิ้น" ไว้ตรงใต้จุดศูนย์ถ่วง และรวมเข้ากับเส้นรอบรูปอะคริลิค
- **การตรวจสอบขนาดฐาน** — ใช้หลักการรูปหลายเหลี่ยมรับน้ำหนัก ตรวจสอบว่าฟิกเกอร์สามารถตั้งได้ด้วยตนเองที่ความกว้างฐานที่กำหนด และร่องเสียบพอดีกับความลึกฐานที่กำหนด
- **การจำลองการล้ม** — คำนวณมุมล้มซ้าย/ขวา/หน้า/หลังเป็น `θ = atan(ระยะจากขอบรับน้ำหนัก / ความสูงจุดศูนย์ถ่วง)`
- **การสร้างเส้นตัด** — สร้างเส้นรอบรูปที่เรียบเนียนผ่านระยะขอบ การเติมช่องว่างแคบ การทำให้เรียบ และการเติมเส้นโค้ง (เบซิเย)
- **การเชื่อมชิ้นส่วนหลายชิ้น** — ชิ้นส่วนที่แยกกันจะไม่ถูกห่อด้วย Convex Hull แต่จะเชื่อมเป็นชิ้นเดียวด้วยจุดเชื่อมที่ตามแนวรูปทรง
- **ตัวอย่างแบบเรียลไทม์** — การแสดงซ้อนทับ ซูมด้วยล้อเลื่อน ลากเลื่อน ไม้บรรทัดขนาดจริง และตารางขนาดจริง (สลับได้ ค่าเริ่มต้นปิด) สามารถสลับเป็น **โหมดตัวอย่างสำเร็จรูป** เพื่อซ่อนเส้นนำสายตา
- **ตัวอย่าง 3D** — ดูผลิตภัณฑ์สำเร็จรูปในแบบ 3 มิติ (อะคริลิคโปร่งใส ภาพบนด้านหลัง / ชั้นขาว ฐานที่มีร่องผ่าน) ตัวเลื่อนเอียงแสดงการล้ม และอนิเมชันแยกชิ้นส่วนแสดงการประกอบของลิ้น พื้นแสดงตารางขนาดจริง (ช่อง 10 มม. ค่าเริ่มต้นเปิด) และสามารถใช้พื้นผิวตัวอย่างลายไม้ที่แถมมาหรืออัปโหลดภาพเองได้
- **ส่งออก SVG / Illustrator (.ai)** — ส่งออกเส้นรอบรูป (ตัว + คอ + ลิ้น) ร่องเสียบ และร่องบนฐานในขนาดจริง (มม.) `.ai` จะฝังภาพ artwork เสมอและแยกเลเยอร์ SVG ส่งออกเฉพาะเส้นข้อมูลตามค่าเริ่มต้น ฝังภาพ artwork เฉพาะเมื่อเลือกตัวเลือก

## วิธีใช้

1. โหลดภาพ PNG (RGBA) ด้วยการลากและวางหรือเลือกไฟล์ ตามค่าเริ่มต้น **α=0 ถือเป็นโปร่งใส และ α>0 ถือเป็นอะคริลิค** (สามารถเปลี่ยนขอบเขตด้วยเกณฑ์อัลฟา)
2. ใส่ความสูงฟิกเกอร์ (มม.) ในแผงซ้าย ค่านี้คือ **ความสูงรวมจากพื้นรอง (ด้านล่างฐาน) ถึงด้านบนของเส้นตัด** จากค่านี้จะได้ "ความสูง artwork (มม.)" โดยหักระยะสูงภายนอก artwork (ระยะขอบเส้นตัด × 2 + ระยะยกแผ่น + ความหนา) และคำนวณ `mm_per_pixel` จากอัตราส่วนของความสูง artwork เป็นมม. ต่อพิกเซล ความสูงของภาพไม่ถูกใช้ ดังนั้นขนาดจริงจะไม่เปลี่ยนแปลงตามปริมาณขอบโปร่งใสของ PNG
3. ปรับแต่งพารามิเตอร์แต่ละรายการ การวิเคราะห์และการวาดใหม่จะทำงานทันทีที่มีการเปลี่ยนแปลง
4. ตรวจสอบผลการวิเคราะห์และมุมล้มในแผงขวา แล้วส่งออก SVG

### โครงสร้างร่องเสียบ (คอ + ลิ้น)

ร่องเสียบประกอบด้วยสี่เหลี่ยมสองชิ้นที่มีความกว้างต่างกัน คอมีความกว้างมากกว่าลิ้น และ **ไหล่ที่เกิดขึ้นวางอยู่บนด้านบนของฐานเพื่อหยุดการเสียบที่ความลึกของลิ้น (= ความหนาแผ่น)**

```
         ┌─────────────────┐
         │   แผ่นอะคริลิค   │   ← เส้นตัด (ตัว)
         └────┬───────┬────┘
              │  คอ   │           ความกว้าง = ความกว้างคอ (> ความกว้างร่อง)
              │       │
   ═══════┌───┴──┐┌───┴───┐═════   ← ไหล่พักบนด้านบนฐาน
   ฐาน    │ ไหล่ ││ ไหล่  │  ฐาน
   ───────┴───┐  └┘  ┌───┴──────  ด้านบนฐาน
              │ ลิ้น  │           ความกว้าง = ความกว้างร่อง (มม.)
              │       │           ความลึก = ความหนาแผ่น (มม.)
              └───────┘
```

ตามการออกแบบ จะต้องสอดคล้องเสมอว่า `ความกว้างคอ ≥ ความกว้างร่อง + 2 × ความกว้างไหล่ขั้นต่ำ (ด้านละ 0.5 มม.)` หากการขยายร่องทำให้ความกว้างคอต่ำกว่าขีดจำกัดนี้ ความกว้างคอจะถูกดันให้สูงถึงค่าขั้นต่ำโดยอัตโนมัติ

### กรณีข้อผิดพลาด

จะไม่มีข้อยกเว้นที่ทำให้แอปล่ม ข้อความจะแสดงใน UI

- **ไม่มีบริเวณอะคริลิค** — ภาพโปร่งใสทั้งหมด หรือเกณฑ์อัลฟาสูงจนไม่มีพิกเซลที่ไม่โปร่งใสเหลืออยู่
- **ไม่สามารถคำนวณสเกลได้** — ความสูงฟิกเกอร์น้อยกว่าหรือเท่ากับความสูงภายนอกของ artwork (ระยะขอบเส้นตัด × 2 + ระยะยกแผ่น + ความหนา) ทำให้ไม่สามารถได้ความสูง artwork
- **ไม่สามารถจัดวางร่องเสียบได้** — การเยื้องมากเกินไป หรือคอยื่นออกนอกแผ่น
- **ไม่สามารถคำนวณฐานได้** — ความกว้างฐานที่กำหนดน้อยกว่าที่จำเป็น (`max(2 × |จุดศูนย์ถ่วง X − ศูนย์กลางร่อง X|, ความกว้างร่อง)`) หรือความลึกฐานน้อยกว่าที่จำเป็น (`ความหนา + 2 × |การเยื้องหน้า-หลัง|`) ฐานจะไม่ถูกขยายโดยอัตโนมัติ ให้ตรวจสอบพารามิเตอร์

## การพัฒนา

```bash
npm install
npm run dev          # เซิร์ฟเวอร์สำหรับพัฒนา
npm run build        # บิลด์ใช้งานจริง (สร้าง dist/)
npm run preview      # ดูตัวอย่างบิลด์ในเครื่อง
npm run lint         # ESLint (ต้องไม่มีคำเตือน)
npm run format       # Prettier
```

เนื่องจากแอปให้บริการภายใต้ชื่อ repository บน GitHub Pages `base` ของ Vite จึงถูกตั้งค่าคงที่เป็น `/daiza/` การ push ไปยัง `main` จะรัน `.github/workflows/deploy.yml` ซึ่ง lint, build และ deploy

### กองเทคโนโลยี

React 19 / TypeScript (strict) / Vite / Tailwind CSS v4 / shadcn/ui / polygon-clipping / pdf-lib (`.ai`) / Three.js + React Three Fiber (ตัวอย่าง 3D) การจัดการสถานะใช้ React Hooks เท่านั้น ไม่ใช้ไลบรารีจัดการสถานะทั่วไปอย่าง Redux

pdf-lib และชุดการพึ่งพา 3D ถูกแยกไว้ใน **dynamic import** จึงจะดาวน์โหลดก็ต่อเมื่อจำเป็น

### สถาปัตยกรรม

UI, การวิเคราะห์ภาพ, การคำนวณฟิสิกส์, การเรนเดอร์ และการส่งออก ถูกแยกออกจากกันอย่างชัดเจน โค้ดภายใต้ `analysis/` เป็นลอจิก pure function ที่ไม่ขึ้นกับ React ความล้มเหลวจะคืนค่าเป็น `null`

```
src/
  components/   LeftPanel / Preview / ResultPanel / ExportPanel / Ruler / Grid … UI (เลย์เอาต์ 3 แผง)
                preview3d/                                  … มุมมอง 3D (R3F, โหลดแบบ lazy)
  analysis/     imageLoader, contour, centroid, distance, slot, base, stability,
                scale, pipeline, analysis.worker            … ลอจิกบริสุทธิ์
  render/       overlay, simulation, ruler, scene3d         … โมเดลการเรนเดอร์ (บริสุทธิ์)
                texture3d                                   … พื้นผิวสำหรับ artwork / ชั้นขาว / พื้น (DOM)
  assets/       textures/wood.png                           … พื้นผิวลายไม้ตัวอย่าง (อ้างอิงเฉพาะชังก์ 3D)
  export/       geometry, svg, ai, raster                   … สร้าง SVG / Illustrator ในขนาดจริง (มม.)
  model/        state.ts, types.ts, errors.ts, pixelStore.ts
  hooks/        useAnalysis, useAppState, useViewport
  utils/        geometry.ts, image.ts, curve.ts
```

เรขาคณิตของฉาก 3D (เส้นตัด → รูปทรงอัด การแปลงเป็นระบบพิกัดมม.) อยู่ใน `render/scene3d.ts` เป็นฟังก์ชันบริสุทธิ์ ไม่ขึ้นกับ Three.js หรือ React จุดกำเนิดคือ **ศูนย์กลางฐานบนพื้นรอง** Y ขึ้นด้านบน Z หน้าไปข้างหน้า โดยด้านล่างของลิ้นอยู่ที่ Y=0 (เสมอกับด้านล่างฐาน)

### ไพพ์ไลน์การวิเคราะห์

เพื่อป้องกันการค้างแม้กับภาพระดับ 3000 px ไพพ์ไลน์ถูกแบ่งเป็น 2 เฟส และทำงานบน Web Worker

- **เฟส 1 `analyzeImage`** — การประมวลผลล่วงหน้าที่ขึ้นกับภาพ (แยกช่องอัลฟา) O(W×H) ทำงานเมื่อเปลี่ยนภาพเท่านั้น แผ่นอัลฟาจะถูกเก็บไว้ภายใน Worker และไม่ถูกส่งไปยังเธรดหลัก การแปลงเป็นค่า 0/1 จะไม่ทำในขั้นตอนนี้ เพราะขึ้นกับพารามิเตอร์เกณฑ์อัลฟา
- **เฟส 2 `runAnalysis`** — การคำนวณที่ขึ้นกับพารามิเตอร์ (แปลงเป็นค่า 0/1 → เส้นตัด → จุดศูนย์ถ่วง → ร่องเสียบ → ฐาน → มุมล้ม) ขั้นตอนหนัก (การแปลงเป็นค่า 0/1 การขยาย EDT การสร้างเส้นตัด การรวมร่องเสียบ) ถูก memoize ด้วยกุญแจที่ขึ้นกับพารามิเตอร์ ดังนั้นการเปลี่ยนแปลงที่ไม่เกี่ยวข้อง เช่น ความกว้างฐาน จะไม่กระตุ้นการคำนวณใหม่

เส้นตัดจะถูกสรุปตามลำดับต่อไปนี้ และขั้นตอนจุดศูนย์ถ่วง การจัดวางร่องเสียบ การคำนวณฐาน การซ้อนทับ และการส่งออก SVG ทั้งหมดจะตามเส้นตัดนี้

**การแยกรูป contour → ระยะขอบ (การขยาย EDT) → เติมช่องว่าง (disc closing) → ลบจุดตัดกันของตัวเอง → การทำให้เรียบ → การเติมเส้นโค้ง → การรวมร่องเสียบ**
