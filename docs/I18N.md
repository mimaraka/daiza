# 多言語対応（i18n）ガイド

本アプリは英語・タイ語・日本語の 3 言語に対応している。翻訳は UI 層の 1 箇所に集約し、解析・幾何計算・エクスポートなどの純粋ロジック層には一切の言語依存を持たせない。

## 対応言語

- `en` — English（フォールバック言語）
- `th` — ไทย
- `ja` — 日本語

## ファイル構成

```
src/locales/
├── types.ts      # Locale 型・言語一覧・localStorage キー
├── index.tsx     # LocaleContext / LocaleProvider / useLocale / useTranslation
├── en.ts         # 翻訳キーの正典（as const）
├── th.ts         # タイ語辞書
└── ja.ts         # 日本語辞書
```

`en.ts` は翻訳キー（`TranslationKey`）の正典である。新しいキーを追加するときはまず `en.ts` に書き、必要に応じて `th.ts`・`ja.ts` にも同じキーを追加する。

## 翻訳の使い方

```tsx
import { useTranslation } from '@/locales';

function SomeComponent() {
  const { t, locale } = useTranslation();

  return (
    <div>
      <h1>{t('app.title')}</h1>
      <p>{t('leftPanel.neckWidth', { min: 10 })}</p>
    </div>
  );
}
```

- `t(key)` で翻訳文字列を取得する。
- プレースホルダ `{ { name } }`（実際には `{{name}}`）を使う場合は第 2 引数に置換値を渡す。
- 選択中の言語コードは `locale` で取得できる。

## 翻訳キーの命名規則

フラットなドット記法で、対象コンポーネントまたは概念を先頭に置く。

```
app.title
app.subtitle
headerActions.privacyTooltip
leftPanel.section.acrylic
leftPanel.figureHeight
baseShape.rect
baseShape.customShort
result.imageSize
preview.toolbar.preview3d
preview3d.tiltFalling
errors.baseCalculationFailed
```

## 新しい言語を追加する手順

1. `src/locales/types.ts` の `Locale` 型と `AVAILABLE_LOCALES` に言語コードを追加する。
2. `src/locales/` 直下に新しい辞書ファイル（例: `fr.ts`）を作る。
   - `import type { TranslationKey } from './index';`
   - `export const fr: Record<TranslationKey, string> = { ... };`
3. `src/locales/index.tsx` の `dictionaries` レコードに追加する。
4. 必要に応じて `index.html` のデフォルト `<title>`・`lang` 属性を調整する。
5. `npm run lint && npm run build && npm run format` を実行して問題がないことを確認する。

## エラーメッセージの翻訳

解析層（`analysis/`）や `model/errors.ts` は日本語／英語の文章を直接持たない。代わりに `AnalysisError` は `kind` コードのみを返し、UI 側で `t(`errors.${error.kind}`)` として翻訳する。

新しいエラー種別を追加するときは、次の両方を行う。

1. `src/model/types.ts` の `AnalysisErrorKind` に種別を追加する。
2. `src/locales/en.ts`・`th.ts`・`ja.ts` に `errors.<kind>` キーを追加する。

## 注意事項

- 翻訳が見つからないキーは、自動的に英語（`en`）の値が使われる。
- プレースホルダの置換値は `string | number` に限定する。複雑なフォーマットが必要な場合はコンポーネント側で整形してから渡す。
- UI 以外（コンソールログ・デバッグ用コメントなど）は翻訳の対象外とする。
