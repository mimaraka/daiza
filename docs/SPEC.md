# Daiza（アクリルフィギュア台座設計ツール） 実装指示書

## 概要

TypeScriptを用いて、PNG画像からアクリルフィギュアの重心を解析し、差込口位置・台座サイズを自動計算するWebアプリケーションを実装してください。

本アプリケーションは**GitHub Pages（github.io）上で公開できる静的Webアプリ**として実装し、サーバーを必要としない構成としてください。

画像解析・幾何計算・SVG生成はすべてブラウザ内で完結させてください。

UIは各種パラメータをリアルタイムに変更でき、結果が即座に反映されることを重視します。

# 入力

条件

- RGBA画像

画像はブラウザ内のみで処理し、外部へ送信してはいけません。

# 重心解析

透明以外を均一密度とみなしてください。

重心は画像モーメントで求めます。

```
Cx = Σx / N

Cy = Σy / N
```

---

# 差込口候補

画像最下部から探索します。

差込口幅が完全に収まる範囲のみ候補としてください。

候補が複数ある場合

```
重心真下に最も近い位置
```

を採用してください。

---

# 台座幅

支持多角形の考え方を採用してください。

最低条件

```
重心が支持範囲内
```

安全率を掛けて

推奨台座幅を計算してください。

---

# 転倒シミュレーション

左右方向について

```
転倒角
```

を計算してください。

計算式

```
θ = atan(支持端距離 / 重心高さ)
```

左右それぞれ表示してください。

# 描画更新

パラメータ変更時は

```
解析
↓

状態更新

↓

再描画
```

を即時実行してください。

Reactの再レンダリングを活用し、不要な再計算は `useMemo` や `useCallback` を利用して最小限に抑えてください。

---

# 状態管理

アプリ全体の状態はReact Hooksで管理してください。

過度なグローバル状態管理ライブラリ（Redux等）は使用しないでください。

---

# エラーハンドリング

以下の場合はUI上へ分かりやすく表示してください。

- PNG読み込み失敗
- 非対応画像
- 透明画像
- 差込口が配置不可
- 台座計算不可

例外によるクラッシュは禁止してください。

---

# パフォーマンス

画像サイズが3000px程度でも快適に動作することを目標としてください。

以下を考慮してください。

- 不要な再描画の抑制
- メモ化
- 必要最小限の画像解析
- オーバーレイのみの再描画

## 設計方針

本アプリケーションは**サーバーレスで動作する静的Webアプリケーション**として実装してください。

- GitHub Pagesへそのままデプロイできること
- 画像・解析データはブラウザ内のみで処理すること
- 外部APIへデータを送信しないこと
- 将来的にPWA化やWebAssemblyへの置き換えが容易な設計とすること
